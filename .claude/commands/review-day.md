# /project:review-day

每日開發結束後，自動整理今天的工作成果。

## 執行步驟

1. 執行 `git log --oneline --since="today"` 查看今天的 commits
2. 執行 `git diff HEAD~[N]..HEAD` 查看所有變更
3. 執行 `git status` 確認工作區狀態

## 輸出格式

請用以下格式整理今天的成果：

```
## Day X 完成情況（YYYY-MM-DD）

### ✅ 已完成功能
- [功能1]：說明做了什麼、為什麼這樣做
- [功能2]：...

### 🔧 技術改進
- [改進點]：說明架構或效能的改動

### ⚠️ 已知限制 / 待解決
- [問題]：說明問題和預計解法

### 代碼統計
- 新增：[檔案] (N 行)
- 修改：[檔案] (改了什麼)
- Commits：N 個

### 明天目標
- [ ] 任務1
- [ ] 任務2
```

4. 詢問用戶是否要將此記錄更新到 `CLAUDE.md` 的「目前狀態」區塊
