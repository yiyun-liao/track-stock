# Skill: Review Day（每日開發成果回顧）

## 概述

整理當日開發進度，輸出結構化日誌。包括完成功能、遇到問題、下一步行動。

## 何時使用

- **日報** — 每天工作結束前執行，記錄日進度
- **Sprint 回顧** — 周末或衝刺結束時回顧本週
- **議題檢討** — 重大改動後回顧影響

## 調用方式

```bash
/project:review-day
```

## 必需資訊

- 當日或當周的 commit 日誌（自動讀取）
- 修改檔案列表（自動讀取）
- （可選）手動補充的成就亮點

## 輸出格式

```markdown
# Day X 開發成果回顧

## 📊 數據概覽
- 完成 commit 數：N
- 修改檔案：M
- 新增行數：L
- 刪除行數：D

## ✅ 完成功能
- [ ] 功能 1 — 描述 + 相關 commit
- [ ] 功能 2 — 描述 + 相關 commit

## 🐛 已修復 Bug
- Bug 1 — 原因 + 解決方案
- Bug 2 — 原因 + 解決方案

## 📝 遇到問題
- 問題 1 — 描述 + 暫時解決方案
- 問題 2 — 描述 + 跟進計畫

## 📚 學習心得
- 心得 1
- 心得 2

## 🎯 下一步行動
- [ ] Action 1 — 優先級高
- [ ] Action 2 — 優先級中
- [ ] Action 3 — 優先級低

## 📌 代碼質量檢查
- [ ] ESLint 通過
- [ ] TypeScript 無錯誤
- [ ] 所有測試通過
- [ ] 可能的性能問題已解決
```

## 依賴規則

- @.claude/rules/code-style.md — 了解 commit 規範

## Agents 參與

- **Frontend Agent** — 檢查 React 元件改動質量
- **Scraper Agent** — 檢查 API 集成改動
- **Analyzer Agent** — 檢查分析邏輯改動

## 常見變化

| 場景 | 輸出內容 |
|------|---------|
| 新增功能日 | 詳細描述新功能、架構決策、遇到的挑戰 |
| Bug 修復日 | 重點強調根本原因、修復方案、預防措施 |
| 重構日 | 強調架構改善、性能提升、技術債清除 |
| 維護日 | 記錄維護內容、依賴更新、配置調整 |

## 自動生成提示

使用 git 日誌自動生成骨架：

```bash
git log --oneline --since="24 hours ago"
git diff --stat HEAD~N
```
